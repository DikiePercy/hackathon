# 🎉 Ollama успешно настроен!

## Что это значит?

Ваша RAG система теперь использует **локальную нейросеть Ollama (llama3:8b)** вместо платного API Gemini.

### Преимущества:
- ✅ **Бесплатно** - не нужен API ключ
- ✅ **Приватность** - все данные остаются на вашем компьютере
- ✅ **Без интернета** - работает полностью оффлайн
- ✅ **Многоязычность** - поддерживает русский, кыргызский, турецкий

## Как пользоваться?

### 1. Откройте веб-интерфейс

- **Основной**: http://localhost
- **Альтернативный**: http://localhost:8501

### 2. Войдите в систему

Используйте логин и пароль администратора из файла `.env`

### 3. Загрузите документы

1. Перейдите в раздел "Документы" или "Управление"
2. Загрузите текстовые файлы (.txt или .md)
3. Система автоматически обработает их

### 4. Задавайте вопросы

1. Перейдите в раздел "Чат"
2. Напишите вопрос на русском, кыргызском или турецком
3. Получите ответ от локальной модели!

## Проверка работы

Запустите быструю проверку:
```bash
./quick_test_ollama.sh
```

Или вручную:
```bash
# Ollama работает?
curl http://localhost:11434/api/tags

# Backend работает?
curl http://localhost:8000/health
```

## Скорость работы

- **Первый запрос**: ~3-5 секунд (модель загружается в память)
- **Последующие**: ~1-3 секунды (модель уже в памяти)

Это нормально! Локальные модели работают медленнее, чем облачные API, но зато бесплатно и приватно.

## Смена модели

Хотите попробовать другую модель?

```bash
# Посмотреть доступные модели
ollama list

# Скачать новую модель (например, Mistral)
ollama pull mistral:7b

# Изменить в .env
echo "RAG_OLLAMA_MODEL=mistral:7b" >> .env

# Перезапустить backend
docker restart hackathon_python
```

## Возврат на Gemini

Если нужно вернуться на Gemini:

```bash
# Измените в .env:
RAG_LLM_PROVIDER=gemini
RAG_EMBEDDING_PROVIDER=gemini
GEMINI_API_KEY=your-api-key

# Перезапустите:
docker restart hackathon_python
```

## Проблемы?

### Медленно работает
- Проверьте использование RAM: `free -h`
- Проверьте CPU: `htop`
- Попробуйте меньшую модель

### Ошибки при ответах
- Проверьте логи: `docker logs hackathon_python`
- Проверьте Ollama: `journalctl -u ollama -f`

### Backend не запускается
- Проверьте статус: `docker ps`
- Проверьте логи: `docker logs hackathon_python`
